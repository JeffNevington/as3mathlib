Summary

I have attempted to create an intuitive, functionality-driven heirarchy to house the library classes. This strategy causes interfaces and their dependent classes to be located across several packages, so I've noted dependent class relationships within the text below to assist in your explorations. This library is under construction, and as it evolves there may be changes to its architecture, so check back often.
