MCE class

version: 1.6

description:

Implements the static behaviour of the MCE Class, a revision of the original class by Ted Patrick, It supports dynamic instantiation, with the ability to bind movieClipDataType to class with arguments. Place MCE.as in the default-level folder of your classpath.

Options for instantiation: 

 1. create an empty mc, optionally passing an init object
 2. attach a library linkage mc, optionally passing an init object
 3. create a new class instance, optionally passing an init object and class arguments
 4. cast an existing library linkage mc to a class, optionally passing an init object and class arguments
Follow this for links to full documentation and discussions of dynamic instantiation.

usage:

Static MCE.attach is called directly.

parameters:

  • None -- no class parameters.

MCE Properties:

none -- no class properties.

MCE Methods:

attach

description: Dynamic MovieClip instantiation with class association.

usage: 

MCE.attach(scope,depth,name[,linkage][,initObj][,class][,classArguments])

parameters:

  • t (MovieClip) -- scope for the new clip.
  • d (Number) -- depth of new clip.
  • n (String) -- name of new clip.
  • l (String) -- library linkage of new clip ... optional.
  • i (Object) -- initObj of new clip ... optional.
  • c (Object) -- class association of new clip ... optional.
  • ca (Array) -- class parameters of new clip ... optional.

returns: (variant) -- return dataType is variant, so there is no method return casting
